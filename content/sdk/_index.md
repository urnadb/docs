传统的关系型数据库使用 SQL 作为数据操作语言，业务层应用一般需要通过 JDBC 及其配套的数据库驱动来执行 SQL 与数据库通信。而 UrnaDB 数据库则采用基于 RESTful + JSON + 官方 SDK 来实现，这也是绝大多数现代分布式数据库的主流模式，为每种编程语言都提供对应的 SDK ，用户只需引入 SDK 即可使用。随着 SDK 的更新，客户端 API 也会同步升级，开发者无需额外学习 SQL 语言即可完成数据操作。SDK 的实现采用用户所在的编程语言编写，因此能够充分利用该语言生态下的静态类型系统和 IDE 工具链，实现更完善的编译期检查与智能提示，确保 SDK API 的正确使用。
